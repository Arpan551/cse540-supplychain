## Project Description

A blockchain-powered supply chain provenance system on Ethereum. Five stakeholder roles — Producer, Distributor, Retailer, Regulator, and Consumer — can transparently track and verify a product's journey from origin to delivery. All critical events are recorded immutably on-chain via three Solidity smart contracts. Large artifacts (certificates, IoT sensor logs) are stored off-chain on IPFS; only content-addressed CIDs land on-chain to minimize gas costs.

## Quick Start — Local Demo

### Prerequisites

- Node.js v18+
- MetaMask browser extension
- Python 3.x (optional, for IoT simulation)

### 1. Install dependencies

```bash
# Root (Hardhat + contracts)
npm install

# Frontend
cd frontend && npm install && cd ..
```

### 2. Start a local Hardhat node

```bash
npx hardhat node
```

Leave this terminal running. It prints 20 pre-funded accounts — you will use accounts 0–4 as admin, producer, distributor, retailer, and regulator respectively.

### 3. Compile and deploy contracts

In a second terminal:

```bash
npx hardhat compile
npx hardhat run scripts/deploy.js --network localhost
```

This prints three contract addresses and saves them to `deployed-addresses.json`.

### 4. Configure the frontend

```bash
cp frontend/.env.example frontend/.env
```

Open `frontend/.env` and paste the two addresses from the deploy output:

```
VITE_PRODUCT_REGISTRY=0x5FbDB2315678af...   # ProductRegistry address
VITE_VERIFICATION_LOG=0xe7f1725E7734CE...   # VerificationLog address
```

### 5. Seed demo data

```bash
npx hardhat run scripts/seed.js --network localhost
```

This sets up roles for five Hardhat signers and creates:

| Product | Batch ID | Journey |
|---|---|---|
| #1 | BATCH-PHARMA-2024-A | Full: Registered → Shipped → InStorage → Delivered + FDA cert |
| #2 | BATCH-FOOD-2024-B | Flagged (simulated recall) |
| #3 | BATCH-LUXURY-2024-C | Registered only (still in production) |

### 6. Run the frontend

```bash
cd frontend && npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

**In MetaMask:** Add a custom network with RPC URL `http://127.0.0.1:8545` and chain ID `31337`. Import the Hardhat test accounts using the private keys printed by `npx hardhat node` to test different stakeholder roles.

### 7. Run tests

```bash
npx hardhat test
```

All 26 tests should pass in under 2 seconds.

### 8. Run simulated IoT data (optional)

```bash
python scripts/simulate_iot.py
```

## Smart Contract Overview

### SupplyChainAccessControl (`contracts/AccessControl.sol`)

Built on OpenZeppelin's `AccessControl`. The deployer gets `DEFAULT_ADMIN_ROLE` and can grant/revoke:

| Role | Constant | Who uses it |
|---|---|---|
| `PRODUCER_ROLE` | `keccak256("PRODUCER_ROLE")` | Manufacturers |
| `DISTRIBUTOR_ROLE` | `keccak256("DISTRIBUTOR_ROLE")` | Logistics / shipping |
| `RETAILER_ROLE` | `keccak256("RETAILER_ROLE")` | Stores / end receivers |
| `REGULATOR_ROLE` | `keccak256("REGULATOR_ROLE")` | Certifying authorities |

### ProductRegistry (`contracts/ProductRegistry.sol`)

| Function | Role required | Description |
|---|---|---|
| `registerProduct(batchId, metadataCID)` | Producer | Mints a new on-chain product |
| `transferCustody(productId, newOwner, note)` | Producer or Distributor (must be current owner) | Hands off custody |
| `updateStatus(productId, newStatus, note)` | Any authorized stakeholder | Logs a status change |
| `confirmDelivery(productId, note)` | Retailer | Marks product as Delivered |
| `getProduct(productId)` | Anyone | Returns current product data |
| `getHistory(productId)` | Anyone | Returns full provenance trail |

**Status enum:** `Registered(0)`, `Shipped(1)`, `InStorage(2)`, `Delivered(3)`, `Flagged(4)`

### VerificationLog (`contracts/VerificationLog.sol`)

| Function | Role required | Description |
|---|---|---|
| `issueCertification(productId, certType, documentCID, expiresAt)` | Regulator | Issues a cert linked to an IPFS document |
| `revokeCertification(certId)` | Regulator | Marks a cert invalid |
| `getCertification(certId)` | Anyone | Returns cert details |
| `getCertificationsForProduct(productId)` | Anyone | Returns all cert IDs for a product |

## Sepolia Testnet Deployment

Create a `.env` file in the project root (see `.env.example`):

```
SEPOLIA_RPC_URL=https://sepolia.infura.io/v3/YOUR_KEY
PRIVATE_KEY=your_wallet_private_key
```

Then:

```bash
npx hardhat run scripts/deploy.js --network sepolia
```

Update `frontend/.env` with the Sepolia addresses and set MetaMask to the Sepolia network.
